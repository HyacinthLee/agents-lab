# 命令行计算器软件规格说明书

## 1. 系统架构设计

本系统采用分层架构设计：表现层（命令行参数解析）负责接收用户输入，业务逻辑层（计算引擎）执行运算并处理异常，通过标准输出返回结果。整体为单进程命令行工具，无外部依赖。

## 2. 模块划分

| 模块 | 职责 |
|------|------|
| `cli` | 解析命令行参数，验证输入参数数量与格式 |
| `parser` | 将字符串输入转换为数值类型，识别运算符 |
| `calculator` | 执行四则运算，实现运算逻辑 |
| `error_handler` | 统一异常处理，输出中文错误提示 |

## 3. 接口定义

### `cli.parse_args(args: list[str]) -> tuple[float, str, float] | Error`
- **输入**：命令行参数列表（如 `["10", "+", "5"]`）
- **输出**：成功返回 `(操作数1, 运算符, 操作数2)`，失败返回 `Error`

### `parser.to_number(s: str) -> float | Error`
- **输入**：数字字符串
- **输出**：成功返回浮点数，失败返回 `Error`

### `calculator.calculate(a: float, op: str, b: float) -> float | Error`
- **输入**：两个操作数及运算符（`+` `-` `*` `/`）
- **输出**：成功返回计算结果，除零等错误返回 `Error`

### `error_handler.format_error(e: Error) -> str`
- **输入**：错误对象
- **输出**：格式化后的中文错误提示字符串

## 4. 数据结构

```python
# 运算符枚举
type Operator = "+" | "-" | "*" | "/"

# 错误类型
class Error:
    code: int          # 错误码
    message: str       # 错误描述

# 计算结果（联合体）
type Result = float | Error
```

## 5. 错误码定义

| 错误码 | 说明 |
|--------|------|
| 1001 | 参数数量错误 |
| 1002 | 非法数字格式 |
| 1003 | 不支持的操作符 |
| 1004 | 除零错误 |

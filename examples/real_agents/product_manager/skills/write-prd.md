# Skill: Write PRD

## 描述
撰写产品需求文档 (Product Requirements Document)

## 输入
- 用户原始需求描述
- 目标用户群体
- 业务背景（可选）

## 输出格式
```markdown
# PRD: [功能名称]

## 1. 功能概述
一句话描述这个功能是什么。

## 2. 用户故事
- 作为 [用户角色]
- 我想要 [功能]
- 以便于 [价值]

## 3. 功能列表

### P0 (必须有)
- [ ] 功能点1: 描述
- [ ] 功能点2: 描述

### P1 (应该有)
- [ ] 功能点1: 描述

### P2 (可以有)
- [ ] 功能点1: 描述

## 4. 验收标准

### AC1: [功能点名称]
**Given** [前置条件]
**When** [操作]
**Then** [预期结果]

## 5. 非功能需求
- 性能: [具体要求]
- 安全: [具体要求]
- 兼容性: [具体要求]

## 6. 界面原型（可选）
文字描述或伪代码

## 7. 开放问题
- [ ] 问题1
```

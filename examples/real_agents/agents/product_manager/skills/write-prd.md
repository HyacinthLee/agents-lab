## Description

编写产品需求文档

## When to Use

当需要为新功能编写 PRD 时使用

## Input

- feature_name: 功能名称
- target_users: 目标用户
- requirements: 功能需求

## Output

- PRD 文档 (Markdown 格式)
- 验收标准

## Steps

1. 理解功能背景和目标用户
2. 编写产品概述
3. 列出功能列表
4. 定义验收标准
5. 保存 PRD 到 workspace

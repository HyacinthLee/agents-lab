# 产品需求文档（PRD）

## 1. 产品概述

### 背景
在日常开发工作中，工程师需要频繁进行数学计算，包括简单的四则运算、科学计算、进制转换等。现有的系统计算器或需要打开图形界面，或功能过于简单，无法满足命令行工作场景下的快速计算需求。一个轻量级、功能完善的命令行计算器能够显著提升开发效率。

### 目标
开发一款命令行计算器工具，支持基本算术运算、科学计算、进制转换、表达式求值等功能，为用户提供快速、准确、便捷的计算体验。

### 目标用户
- 软件开发工程师
- 系统运维人员
- 数据分析师
- 需要在命令行环境下进行计算的技术人员

## 3. 验收标准

| 场景 | 操作 | 期望结果 |
|------|------|---------|
| 基本加法 | 输入 `calc 1 + 2` | 输出 `3` |
| 混合运算 | 输入 `calc "(1 + 2) * 3"` | 输出 `9` |
| 浮点运算 | 输入 `calc 3.14159 * 2` | 输出 `6.28318` |
| 除零保护 | 输入 `calc 1 / 0` | 输出错误提示 `Error: Division by zero` |
| 幂运算 | 输入 `calc 2 ^ 10` | 输出 `1024` |
| 开方运算 | 输入 `calc sqrt(16)` | 输出 `4` |
| 三角函数 | 输入 `calc sin(3.14159/2)` | 输出约等于 `1` |
| 进制转换 | 输入 `calc 0b1010 to dec` | 输出 `10` |
| 十六进制转二进制 | 输入 `calc 0xFF to bin` | 输出 `0b11111111` |
| 复杂表达式 | 输入 `calc "(100 + 200) / 3 - 50"` | 输出 `50` |
| 交互模式 | 输入 `calc -i` | 进入交互式提示符 `calc>` |
| 交互模式退出 | 在交互模式下输入 `exit` 或按 `Ctrl+C` | 正常退出程序 |
| 历史查看 | 输入 `calc --history` | 显示最近计算记录列表 |
| 帮助信息 | 输入 `calc --help` | 显示完整的使用说明 |
| 无效输入 | 输入 `calc abc` | 输出错误提示 `Error: Invalid expression` |
| 语法错误 | 输入 `calc 1 + * 2` | 输出错误提示 `Error: Syntax error` |

## 4. 使用方式

### 命令行直接计算
```bash
calc <表达式>
```

### 交互模式
```bash
calc -i
# 或
calc --interactive
```

### 查看历史记录
```bash
calc --history
```

### 查看帮助
```bash
calc --help
```

### 从文件批量计算
```bash
calc -f calculations.txt
```

### 使用示例
```bash
# 基本运算
calc 100 + 200
calc "3.14 * 2 ^ 2"

# 科学计算
calc "sin(30) + cos(60)"
calc "log(100, 10)"

# 进制转换
calc 255 to hex      # 输出 0xFF
calc 0x1A to bin     # 输出 0b11010
```

## 5. 非功能需求

### 性能要求
- 简单四则运算响应时间 < 100ms
- 复杂表达式计算响应时间 < 500ms
- 启动时间 < 200ms
- 内存占用 < 50MB

### 兼容性要求
- 支持 Linux、macOS、Windows 操作系统
- 支持 Python 3.8 及以上版本
- 不依赖外部图形界面库
- 支持主流终端模拟器（xterm、iTerm2、Windows Terminal 等）

### 安全要求
- 对用户输入进行严格校验，防止代码注入
- 禁用危险函数执行（如 `eval` 的直接使用）
- 表达式解析采用白名单机制，仅允许预定义的运算符和函数
- 历史记录文件权限设置为仅当前用户可读写（600）
- 不收集或上传任何用户数据

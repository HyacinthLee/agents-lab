## 2. 测试策略

### 测试方法
- **黑盒测试**：基于PRD验收标准，验证输入输出行为
- **白盒测试**：检查表达式解析器内部逻辑分支（如运算符优先级处理）

### 测试优先级
| 优先级 | 覆盖范围 | 用例编号 |
|--------|----------|----------|
| P0 | 核心功能，阻塞发布 | TC-001~TC-005, TC-011~TC-012 |
| P1 | 重要功能，影响体验 | TC-006~TC-008, TC-013~TC-014 |
| P2 | 次要功能，优化项 | TC-009~TC-010, TC-015 |

### 总测试用例数：15 个

## 4. 测试通过标准

### 测试覆盖率要求
- P0级用例：100% 执行通过
- P1级用例：≥90% 执行通过
- P2级用例：≥80% 执行通过

### 通过率要求
- 总体测试用例通过率：100%（15/15）
- 阻塞性缺陷（P0）：0个
- 严重缺陷（P1）：≤1个
- 一般缺陷（P2）：≤2个

### 性能要求
| 指标 | 要求 | 验证方法 |
|------|------|----------|
| 简单运算响应时间 | < 100ms | 使用`time`命令测量 |
| 复杂表达式响应时间 | < 500ms | 使用`time`命令测量 |
| 启动时间 | < 200ms | 使用`time`命令测量 |
| 内存占用 | < 50MB | 使用系统监控工具 |

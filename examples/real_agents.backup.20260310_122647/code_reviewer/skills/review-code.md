# Skill: Review Code

## 描述
系统性地审查代码质量

## 审查维度
1. **正确性**: 代码是否实现了预期功能
2. **可读性**: 命名、结构、注释
3. **可维护性**: 耦合度、重复代码
4. **测试**: 覆盖率、测试质量
5. **安全**: 常见漏洞模式
6. **性能**: 明显瓶颈

## 审查流程
1. 阅读 PRD/规格，理解预期功能
2. 快速浏览整体结构
3. 逐文件深入审查
4. 记录问题（标记严重程度）
5. 总结评价和建议

## 严重程度定义
- **High**: 必须修复（bug、安全漏洞）
- **Medium**: 建议修复（可读性、设计问题）
- **Low**: 可选（风格、优化）

## 输出
```markdown
## 审查报告: [项目/文件]

### 元信息
- 审查日期: YYYY-MM-DD
- 审查范围: [文件列表]
- 总体状态: [APPROVED / NEEDS_WORK]

### 统计
- 代码行数: X
- 测试覆盖率: X%
- 问题数: High(0) Medium(0) Low(0)

### 详细发现
[按严重程度排序]

### 建议
[按优先级排序]
```
